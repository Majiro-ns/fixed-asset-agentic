### 1-3. 録画ソフト設定

#### OBS Studio（推奨）

| 設定項目 | 推奨値 |
|---------|--------|
| 出力解像度 | 1920 x 1080（FHD） |
| FPS | 30fps |
| エンコーダ | x264 または NVENC |
| ビットレート | 4000-6000 kbps（映像）|
| 音声ビットレート | 128 kbps |
| 出力形式 | MP4（.mp4） |
| 録画モード | ウィンドウキャプチャ（Chrome指定） |

#### OBS 設定手順

1. **映像ソースの追加**
   - ソース → 「+」 → 「ウィンドウキャプチャ」
   - ウィンドウ: Chrome のデモタブを選択
   - 「カーソルをキャプチャ」にチェック

2. **音声ソースの追加**
   - ソース → 「+」 → 「音声入力キャプチャ」
   - マイクを選択（外付けマイク推奨）
   - デスクトップ音声はミュートに

3. **録画設定**
   - 設定 → 出力 → 録画
   - 録画パス: デスクトップ等のわかりやすい場所
   - 録画フォーマット: mp4

4. **テスト録画**
   - 10秒ほどテスト録画して再生確認
   - 音声レベル（-20dB ～ -6dB が適正範囲）
   - 映像のカクつきがないか確認

#### マイク設定

| 項目 | 推奨 |
|------|------|
| マイク種類 | ヘッドセットまたは外付けコンデンサーマイク |
| 環境音 | エアコン・ファンの音が入らない静かな環境 |
| マイク位置 | 口元から20-30cm |
| ポップガード | あれば使用（息がかかる音を防止） |
| テスト | 録画前に必ず音声テストを実行 |

### Scene 3: Case 2 -- GUIDANCE: 止まる→聞く→変わる→DIFF（0:40 - 2:10）

**デモの核心部分。ここに最も時間を割く。**

#### Scene 3-1: 止まる（0:40 - 0:55）

##### 画面操作

| 時間 | 操作 |
|------|------|
| 0:40 | PDFアップロードエリアに戻る（ブラウザを上にスクロール） |
| 0:42 | `demo04_guidance_ambiguous.json`（撤去・移設混在工事）のボタンをクリック |
| 0:45 | スケルトンローディング表示 |
| 0:47 | GUIDANCE 結果が表示される |
| 0:47-0:55 | 黄色のGUIDANCEパネルをじっくり見せる（**ここで1-2秒の間を取る**） |

##### ナレーション

> **[0:40-0:47]**
> 次が核心です。今度は曖昧な見積書を入れます。古い設備の撤去と新しい設備の移設が混在した工事見積書。資産か経費か、人でも迷うケースです。

> **[0:47-0:55]**
> （間を取って）GUIDANCE。**AIが自ら判断を停止しました。**「判断が割れるから、自分では決めない」という選択。これが Stop-first 設計の核心です。

##### 画面上の見どころ

- 「🛑 AIが判断を保留しました」ヘッダー（黄色カード、大きく表示）
- 「この書類は自動判定できません。下のボタンで教えてください。」サブテキスト
- プログレス表示: **質問1（現在）** → 質問2 → 判定

> **[1:05-1:15]**

| クロージング | 20秒以内 |

> 3分を超える場合は、Case 1 を短縮するか、技術ハイライトを圧縮する。

### 4-2. 音声の明瞭さ

| チェック項目 | 確認方法 |
|------------|---------|
| ナレーションが聞き取れるか | ヘッドフォンで通し再生 |
| 環境音が入っていないか | 静かな部分で雑音チェック |
| 音量が一定か | 冒頭と中盤で音量差がないか |
| 言い淀みがないか | 気になる箇所があればリテイク |
| 話す速度 | 150語/分 程度（速すぎず遅すぎず） |

### 4-3. 画面の見やすさ

| チェック項目 | 確認方法 |
|------------|---------|
| 文字が読めるか | スマートフォンの小さい画面でも再生確認 |
| UIの色が正しいか | 緑（CAPITAL_LIKE）、黄（GUIDANCE）、紫（DIFF）が映えているか |
| カーソルの動きが自然か | 急な動きや迷いがないか |
| スクロールが滑らかか | カクつきがないか |
| ローディング表示が映っているか | スケルトンアニメーション → 結果表示の流れが見えるか |
| 通知の映り込みがないか | Windows通知やアプリ通知が映っていないか |

### 4-4. 内容の正確性

| チェック項目 | 確認方法 |
|------------|---------|
| 判定結果が期待通りか | Case 1: CAPITAL_LIKE、Case 2: GUIDANCE → CAPITAL_LIKE |
| DIFFが正しく表示されたか | Before/After が正しい |
| エラーが映っていないか | コンソールエラー、ネットワークエラー等 |
| デモデータが架空データであること | 実在の企業名等が映っていないか |

### 4-5. ファイル出力

| チェック項目 | 基準 |
|------------|------|
| ファイル形式 | MP4 |
| ファイルサイズ | 提出先の制限を確認（通常 100MB 以下） |
| ファイル名 | わかりやすい名前（例: `fixed_asset_agentic_demo_v1.mp4`） |
| バックアップ | 2箇所以上に保存 |

## 5. トラブルシューティング

### 撮影中に起こりうる問題と対処

| 問題 | 対処法 | 予防策 |
|------|--------|--------|

| DIFF が表示されない | 再判定ボタンの押し忘れを確認 | リハーサルで手順を体に覚えさせる |
| 録画ファイルが破損 | OBS を再起動して再撮影 | テスト録画で正常出力を確認済みにしておく |
