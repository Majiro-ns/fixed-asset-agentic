## Q1: 結局、どのツールで動画を撮るべきか？

### 100点回答

**OBS Studio** を使われよ。

**理由**:
1. **無料**（オープンソース）
2. **プロ品質**（YouTuberの90%が使用）
3. **YouTube最適設定**が可能
4. **ノイズ除去フィルタ内蔵**

### インストール・設定手順（5分で完了）

**Step 1**: ダウンロード
```
https://obsproject.com/ja/download
```
→ 「Windows」をクリック → インストーラ実行

**Step 2**: 初回起動時の設定ウィザード
- 「録画のために最適化」を選択
- 解像度: 1920×1080
- FPS: 30

**Step 3**: 録画設定の調整
1. 設定 → 出力 → 出力モード「詳細」
2. 録画タブ:
   - 録画フォーマット: **mp4**
   - エンコーダ: **x264**（GPUあれば NVENC）
   - レート制御: CBR
   - ビットレート: **6000**

**Step 4**: シーン・ソース設定
1. ソース → 「+」→「ウィンドウキャプチャ」
2. Chrome（Streamlit UI）を選択
3. ソース → 「+」→「音声入力キャプチャ」
4. マイクを選択

**Step 5**: テスト録画
- 「録画開始」→ 10秒話す →「録画停止」
- 保存先フォルダで確認

## Q2: OBS以外の選択肢は？

### 簡単さ重視なら: Windows Game Bar

**操作方法**:
1. Chromeでデモ画面を開く
2. `Win + G` でGame Bar起動
3. 録画ボタンをクリック
4. 終了時は `Win + Alt + R`

**メリット**: インストール不要、超簡単
**デメリット**: 細かい設定不可、音質やや劣る

### クラウド保存なら: Loom

**操作方法**:
1. Chrome拡張をインストール
2. 録画ボタンをクリック
3. 終了後、自動でクラウドに保存

**メリット**: 共有が楽（URLで即共有）
**デメリット**: 無料版は5分まで、編集機能なし

## Q3: 顔出しすべきか？

### 100点回答: **顔出しなし** を推奨

**理由**:
1. **3分という短時間**では情報を絞るべき
2. 審査員は**デモ内容**に集中したい
3. 顔が入ると**気が散る**リスク
4. 過去受賞作品も**操作画面のみ**が主流

**もし顔出しするなら**:
- 右下に小さいワイプ（480×270）
- オープニング/クロージングのみ
- デモ中は非表示

---

## Q4: マイクはどうすればいい？

### 100点回答

**優先順位**:
1. **USB外付けマイク**（あれば最良）
2. **ヘッドセット**（通話用でOK）
3. **PC内蔵マイク**（最終手段）

**内蔵マイク使用時の注意**:
- **静かな部屋**で録音
- OBSで**ノイズ抑制フィルタ**を追加
  - 音声ミキサー → マイク → 歯車 → フィルタ → 「+」→ 「ノイズ抑制」

**音量目標**: ピーク時に -6dB 〜 -3dB（OBSメーターで確認）

## Q5: 何回撮り直してもいいか？

### 100点回答: **何回でもOK、3〜5テイク推奨**

**理由**:
- 編集で最良の部分を繋げられる
- 1発撮りにこだわる必要なし
- プロのYouTuberも平均5テイク

**効率的な撮り方**:
1. **通し撮り**（台本見ながらでOK）
2. ミスした箇所だけ**部分撮り直し**
3. 編集で繋ぐ

---

## Q6: 編集は必要か？ツールは？

### 100点回答: **軽い編集推奨**

**最低限やるべき編集**:
1. 冒頭/末尾の無音部分カット
2. 操作ミス・やり直し部分カット
3. 長いローディング（1秒以上）カット

**推奨編集ツール**:

| ツール | 特徴 | 難易度 |
|--------|------|--------|
| **Clipchamp** | Windows標準、無料 | ★☆☆ |
| **DaVinci Resolve** | プロ級、無料 | ★★★ |
| **Premiere Pro** | 業界標準、有料 | ★★★ |

**最速**: Clipchamp（スタートメニューから起動可能）

## Q8: 動画の長さは厳密に3分以内？

### 100点回答: **2分45秒〜2分59秒が理想**

**理由**:
- 3分ジャストだと**余裕がない印象**
- 2分45秒あたりで終わると**スマートな印象**
- 3分超えは**ルール違反リスク**

**タイムキーパー方法**:
- 録画中、**別デバイスでタイマー表示**
- または編集時にタイムライン確認

---

## Q9: BGMは入れるべきか？

### 100点回答: **入れない** を推奨

**理由**:
1. **声が聞き取りづらくなる**リスク
2. デモ動画は**内容勝負**
3. 著作権問題のリスク回避

**もしBGMを入れるなら**:
- オープニング/クロージングのみ（各5秒）
- デモ中は**無音**
- 著作権フリー音源（YouTube Audio Library）

## Q11: 失敗したらどうする？

### 100点回答: **撮り直せばいい**

**リカバリー方法**:
1. **操作ミス** → その場で「失礼しました」と言って続行、後で編集カット
2. **Cloud Run応答なし** → 1分待って再試行、それでもダメならDocker版フォールバック
3. **声が小さかった** → 編集で音量調整可能

**フォールバック（Cloud Run不調時）**:
```powershell
cd C:\Users\owner\Desktop\fixed-asset-ashigaru
docker build -t fixed-asset-api .
docker run --rm -e PORT=8080 -p 8080:8080 fixed-asset-api
```
→ UIのサイドバーで接続先を `http://localhost:8080` に変更
