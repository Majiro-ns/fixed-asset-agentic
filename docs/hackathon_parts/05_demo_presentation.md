## 2. デモシナリオ分析

### DEMO_JP.md の構造評価

```
セクション1: タイトル＋キャッチコピー（〜10秒）
  → 「知らないを、知っているAI」— インパクトあり ✅
セクション2: 課題提示（〜30秒）
  → 新入社員が固定資産判定で困る場面 ✅
セクション3: 解決策＝GUIDANCE（〜60秒）
  → 「止まる→聞く→変わる」のAgentic設計説明 ✅
セクション4: デモ実演（〜90秒）
  → PDF操作→結果表示→不確実時の質問→再判定 ✅
セクション5: まとめ（〜30秒）
  → 価値の再確認 ✅
```

**ストーリー評価**: 「課題→解決→価値」の流れは**明確**。

### 「止まる→聞く→変わる」の伝達力

DEMO_JP.mdではGUIDANCEの3ステップを説明しているが、以下の課題あり：

1. **視覚的証拠がない** — スクリプトだけで画面を見せられない
2. **Before/Afterが不明確** — 従来のAI（確信度100%で間違える）との比較がない
3. **「聞く」の衝撃が薄い** — AIが「止まって質問する」瞬間が最大のインパクトだが、テキスト説明のみ

### DEMO.md（英語版）との差異

- 英語版は4分（日本語版は3分）— 英語版はやや冗長
- 英語版はナレーション・強調ポイントが細かく、プレゼン補助としては優秀
- 日本語版はコンパクトだが、YouTube動画に必要な「見せ場」の演出が不足
