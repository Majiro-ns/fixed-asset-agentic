## 過去受賞作品の勝ちパターン（調査結果）

### 共通する成功要素

1. **冒頭15秒で問題を数値化**
   - 「教師は1日3時間を事務作業に」
   - 「営業は80%の時間をリード調査に」

2. **「Agentic」の可視化**
   - 複数エージェントのログが同時に流れる
   - 「AIが考えている」ことを画面で見せる
   - 処理の自律性を対比（従来30分→AI3秒）

3. **Before/Afterの劇的対比**
   - ユーザーの1回の入力で複数ステップが自動進行

### 避けるべき失敗パターン

- スライドのみ（デモなし）
- ダミーデータ（test, xxxx）
- 技術用語の羅列
- エラー画面やロード待ち

## 本プロジェクトの勝ちポイント設計

### 差別化要素：「止まるAI」

他の参加者との差別化ポイント：
- **他**: 「AIが自動で判断」（高速・効率）
- **我々**: 「AIが止まって聞く」（安全・信頼）

**審査員視点での価値**:
- 李碩根（松尾研）→ 学術的に正しいアプローチ（不確実性の明示）
- 渡部陽太（CTO）→ エンタープライズ要件対応（監査証跡）
- 中井悦司（Google）→ Responsible AI の実践

### 黄金の3フレーズ

| 瞬間 | フレーズ | 審査員への効果 |
|------|----------|----------------|
| **止まる** | 「判断が割れたら自動処理を停止」 | 安全設計への信頼 |
| **聞く** | 「何が不足し、なぜ必要かを説明」 | 対話型AIの価値 |
| **変わる** | 「追加情報で判定変化、履歴が残る」 | 監査対応力 |

## 動画構成（180秒タイムライン）

### プレゼン専門家推奨構成

```
[0-15秒]  問題提起（フック）
[15-90秒] メインデモ（3ケース）
[90-165秒] Agenticループの可視化（最重要）
[165-180秒] 技術＋クロージング
```

### 秒単位詳細台本

#### 0:00-0:15 オープニング【フック】

**画面**: 黒背景にテキスト or 登壇者の顔

**セリフ**（ゆっくり、明瞭に）:
```
「月末の経理処理。見積書の山を前に、
 『これは固定資産？経費？』と悩んだ経験はありませんか。

 判断を間違えれば、監査で指摘される。
 でも繁忙期に1件1件確認する余裕はない。

 このシステムは、判断が割れる場面で『止まる』設計にしています。」
```

**ポイント**:
- 最初の一言で「経理の現場」を想起させる
- 「監査で指摘」で恐怖訴求（問題の重大性）
- 最後に解決策の核心「止まる設計」を予告

**3つの魅せる瞬間を確実に撮る**

**演出ポイント**:
- カーソルでテキストをなぞる

1. クイックボタン「新規購入・設備増強」をクリック

「判定が変わりました。
 『要確認』から『資産計上の可能性あり』へ。

**演出ポイント**:
- Before→Afterの差分を明確に見せる

## 撮影ツール・設定【決定版】

### 推奨ツール: OBS Studio（無料）

**理由**:
- 完全カスタマイズ可能
- YouTube/Vimeo最適設定
- ノイズ除去フィルタ内蔵

### OBS設定値

| 項目 | 設定値 | 理由 |
|------|--------|------|
| 出力解像度 | 1920×1080 | YouTube推奨 |
| フレームレート | 30fps | デモには十分、ファイルサイズ抑制 |
| エンコーダ | x264 or NVENC | 互換性最高 |
| ビットレート | 6000 Kbps | 品質と容量のバランス |
| 出力形式 | MP4 | 全プラットフォーム対応 |

### OBS設定手順

1. **設定 → 出力 → 出力モード「詳細」**
2. **録画タブ**:
   - 録画フォーマット: mp4
   - エンコーダ: x264（CPUのみ）or NVENC（GPUあり）
   - レート制御: CBR
   - ビットレート: 6000 Kbps
3. **映像タブ**:
   - 基本解像度: 1920×1080
   - 出力解像度: 1920×1080
   - FPS: 30
4. **音声タブ**:
   - サンプルレート: 48kHz

### マイク設定

**推奨**: USB外付けマイク or ヘッドセット

**OBSフィルタ追加**:
1. 音声ミキサー → マイク → 歯車 → フィルタ
2. 「+」→「ノイズ抑制」追加
3. 方式: RNNoise（推奨）

**音量目標**: ピーク時 -6dB 〜 -3dB

### 画面レイアウト

**推奨**: 顔出しなし、デモ画面のみフルスクリーン

**理由**:
- 審査員はデモ内容に集中したい
- 3分という短時間では画面要素を減らすべき
- 顔出しは「信頼感」より「気が散る」リスク

### 代替ツール（OBS困難な場合）

| ツール | 特徴 | 推奨度 |
|--------|------|--------|
| Windows Game Bar | Win+G、最も簡単 | 〇 |
| Loom | ブラウザ拡張、クラウド保存 | 〇 |
| PowerPoint録画 | スライド併用時 | △ |

## 話し方のコツ

### スピード

- **通常**: 150語/分（やや遅め）
- **重要箇所**: さらに遅く
- **特に「止まりました」**: 1秒間の沈黙を入れる

### 声のトーン

| 場面 | トーン |
|------|--------|
| 問題提起 | 低め、落ち着いた声 |
| 操作説明 | 普通、明瞭に |
| 「止まりました」 | **低く、ゆっくり** |
| Agenticの本質 | やや高く、熱を込めて |
| クロージング | 落ち着いた、締めの声 |

### 戦略的沈黙

- GUIDANCE表示後: **2秒の沈黙**
- 「止まりました」の後: **1秒の沈黙**
- 重要フレーズ前: **0.5秒の沈黙**

## 編集ポイント（撮影後）

### カットすべき箇所

- 操作ミス、やり直し
- 長いローディング（1秒以上）
- 「えー」「あのー」

### 残すべき箇所

- GUIDANCE表示後の沈黙
- Beforeの画面（1秒）→ Afterの画面（2秒）

### 推奨編集ツール

- ビットレート: 6000 Kbps
