## 勝ちパターン分析

### Pattern 1: Human-in-the-Loop 設計

**全受賞作品に共通する最重要パターン**

| 作品 | Human-in-the-Loop の実装 |
|------|--------------------------|
| eCoino | 対話型AIでシナリオ比較・対策立案を**人間が選択** |
| FlatJam | 知見・技法の候補を提示し、**人間が吟味・選択**して作曲 |
| Fukushia | 会話ログから自動入力し、**人間が確認・修正** |

**共通点**: AIが全自動で処理するのではなく、**人間の意思決定を支援**する設計

> **本プロジェクトへの適用**:
> Stop-first設計（GUIDANCE）は Human-in-the-Loop そのもの。
> 「AIが止まる」ことで人間の判断を促す設計は、受賞作品と同じ思想。
> **この点を強調すべき。**

---

### Pattern 2: 専門職の業務負荷軽減

**対象ユーザーが明確で、具体的な業務課題を解決**

| 作品 | 対象ユーザー | 解決する業務課題 |
|------|-------------|------------------|
| eCoino | SCM担当者 | サプライチェーン計画修正（想定外のズレ対応） |
| FlatJam | 音楽教師・生徒 | 作曲指導（創造性の引き出し） |
| Fukushia | 社会福祉士 | アセスメント作成（1-2時間/人 → 短縮） |

**共通点**: 「時間がない専門職」の**余裕のなさ**を解消

> **本プロジェクトへの適用**:
> 経理担当者の「月末・決算期の時間的余裕のなさ」は同じ構造。
> README.mdに書いてある「疑う余裕がない現場」は正しいアプローチ。
> **業務時間削減の具体的数値があるとより強い。**

---

### Pattern 3: セレンディピティ（偶発的発見）の設計

**FlatJamの受賞理由から抽出**

- 「セレンディピティ欄」で予期しない知見・技法を提案
- ユーザーが入力した言葉に対して、**AIが選択肢を広げる**
- 単なる自動化ではなく、**人間の思考を拡張**

> **本プロジェクトへの適用**:
> 現状: GUIDANCE時に `missing_fields` と `why_missing_matters` を提示
> 改善案: 「この情報があれば判断が変わる可能性がある」という示唆を追加
> → 経理担当者に新たな視点を提供

---

### Pattern 4: 既存業務フローへの統合

**最優秀賞作品は全て「既存業務に組み込める」設計**

| 作品 | 統合方法 |
|------|----------|
| eCoino | Cloud Run functions連携、既存SCMシステムとの接続 |
| FlatJam | 学校の音楽授業にそのまま導入可能 |
| Fukushia | 既存のアセスメント業務フローをそのまま効率化 |

**共通点**: 新しいワークフローを強制しない

> **本プロジェクトへの適用**:
> 見積書PDF → Opal抽出 → 判定 → 要確認行の可視化
> → 既存の経理業務フローに**追加負担なく**組み込める設計
> **この点をデモで強調すべき。**

---

### Pattern 5: Google Cloud サービスの深い活用

**上位入賞作品のGCP活用パターン**

| サービス | 活用例 | 使用作品 |
|----------|--------|----------|
| Vertex AI / Gemini | LLM基盤 | 全作品 |
| Vertex AI Agent Builder | エージェント構築 | eCoino |
| Cloud Run | API / バックエンド | 全作品 |
| BigQuery | データ分析・格納 | eCoino, FlatJam |
| Vertex AI RAG Engine | 社会資源RAG | Fukushia |
| Firebase Hosting | フロントエンド | Fukushia |

**共通点**: 単一サービスではなく**複数サービスの組み合わせ**

> **本プロジェクトへの適用**:
> 現状: Document AI + Vertex AI Search + Cloud Run
> → 十分な組み合わせ。**活用の深さを技術説明でアピール**すべき。

---

### Pattern 6: 対話型インターフェース

**全受賞作品が対話型UI/UXを採用**

| 作品 | 対話の形式 |
|------|-----------|
| eCoino | 自然言語でシナリオ質問・比較 |
| FlatJam | 自然言語で作曲意図を入力 |
| Fukushia | 利用者との会話ログを解析 |

**共通点**: コマンドライン的UIではなく**自然言語対話**

> **本プロジェクトへの適用**:
> 現状: GUIDANCE時に `questions` で質問を提示、`answers` で回答受付
> → 5ステップのAgenticループ（止まる→根拠提示→質問→再実行→差分保存）
> **これは対話型設計として十分。デモで見せるべき。**

## 本プロジェクトへの具体的提案

### 強化すべき点

1. **Human-in-the-Loop の強調**
   - 「AIが止まる」= Human-in-the-Loop のバリエーション
   - 受賞作品と同じ設計思想であることをピッチで明示

2. **業務時間削減の数値化**
   - 例: 「月末の見積書確認作業 100件 × 5分 = 500分 → GUIDANCE対象のみ確認で80%削減」
   - 具体的な数値があると審査員に響く

3. **対話ループのデモ**
   - 5ステップのAgenticループ（止まる→根拠提示→質問→再実行→差分保存）を**動画で見せる**
   - Before/After の差分表示が最も印象的

### 差別化ポイント

| 他の受賞作品 | 本プロジェクト |
|-------------|---------------|
| AIが**提案**する | AIが**止まる** |
| 人間が**選択**する | 人間が**確認**する |
| 効率化 | 誤判定防止 |

→ 「止まる」ことを価値とする設計は**ユニーク**。これを前面に。

## まとめ: 勝つための3つの鍵

1. **Human-in-the-Loop**: AIが全自動ではなく、人間の意思決定を支援
2. **専門職の余裕のなさを解消**: 時間がない現場の具体的課題を解決
3. **GCPの深い活用**: 複数サービスの組み合わせで技術力を示す

**本プロジェクトは3つ全てを満たしている。自信を持ってピッチすべし。**
