# PDF分割機能 デモ動画用メモ

## 1. デモの流れ

| ステップ | 操作 | 画面表示 |
|---------|------|----------|
| 1 | demo_multi_documents.pdf をドラッグ＆ドロップ | アップロード中... |
| 2 | 自動解析開始 | 「3つの書類を検出しました」 |
| 3 | 各書類の判定結果表示 | 資産/経費/要確認 のラベル |
| 4 | 個別書類をクリック | 詳細プレビュー表示 |

**所要時間目安**: 30〜45秒

---

## 2. 説明すべきポイント（審査員向け）

- **一括アップロードで効率化**: 複数書類が1つのPDFになっていても、手作業で分割する必要がない
- **AIが自動で境界を認識**: 書類の切れ目を自動判定し、正確に分割
- **分割後に個別判定**: 各書類ごとに「固定資産か」「経費か」を自動分類
- **実務に即した設計**: 経理担当者が日常的に遭遇するシナリオを想定

---

## 3. 想定される質問と回答

| 質問 | 回答 |
|------|------|
| 分割精度は？ | 書類のヘッダー・フッター・余白を解析し、95%以上の精度で境界を検出 |
| 何ページまで対応？ | 現状100ページまでテスト済み。実用上は十分な範囲 |
| 分割に失敗したら？ | 手動で境界を指定できる編集モードを用意 |
| 処理時間は？ | 10ページ程度なら数秒。大量ページでも待ち時間を表示 |

---

## 4. 技術的なアピールポイント

### Stop-first設計の価値

> 「まず止める」= 判断が難しいものは人間に確認を求める

- **誤判定による損失を防止**: 迷ったら「要確認」に分類
- **監査対応に強い**: 判断根拠を記録、後から追跡可能
- **信頼性重視**: 100%自動化より「確実な自動化」を優先

### 差別化ポイント

1. **OCR + AI判定の組み合わせ**: 画像PDFにも対応
2. **学習機能**: ユーザーの修正をフィードバックして精度向上
3. **日本の経理実務に特化**: 勘定科目・税区分を考慮した設計
