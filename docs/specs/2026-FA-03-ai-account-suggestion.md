業務目的（Why）

固定資産の判定業務では、
「資産か費用か」「どの勘定科目に近いか」を
人がゼロから考える負荷が大きい。

PDFから抽出された情報と、
ルールベースで整理された中間データをもとに、
AIが 勘定科目・資産区分の候補案を提示することで、

初期判断の時間短縮

判断漏れ・思い込みの防止

新人・非専門者でも検討を開始できる状態の提供

を実現したい。

※ 最終判断・確定は 必ず人が行う。

背景・補足

すでに以下は実装済／前提とする

PDF抽出（事実データ）

ルールベースによる事前整理（資産候補／費用候補など）

AIは 決定しない

AIの役割は
「制度的にあり得る案を、理由付きで並べること」

対象範囲（Scope）
入力（前段の成果物）

PDF抽出結果（extraction）

ルールベース整理結果（preclassification）

以下を含む構造化データ

取引内容要約

金額

購入目的／用途

一括／分割の兆候

資産性に関するフラグ

出力（本SPECで定義）

AIによる 勘定科目・資産区分の候補案リスト

出力仕様（重要）
出力は「候補リスト形式」
{
  "ai_suggestions": [
    {
      "account_category": "固定資産",
      "account_name": "工具器具備品",

    {
      "account_category": "費用",

}

AI案提示ルール（設計上の制約）
MUST（必須）

複数案を出す（1案のみは禁止）

各案に 理由（reason）を必ず付与

confidence は以下のみ使用

high / medium / low

人が否定・修正できる表現にする

MUST NOT（禁止）

勘定科目を確定する表現

「〜である」「確定する」などの断定

税法・耐用年数の最終判断

金額基準だけで結論を出すこと

人の作業を前提としたUI連携意図（参考）

UIでは以下を想定

AI案を 一覧表示

confidence を視覚的に区別

人が「採用／却下／保留」を選択

AI案は ログとして保存

後日の監査・説明用

注意事項（固定）

税務・会計判断の定義変更は行いません

実装範囲は本SPEC記載内容のみ

不明点がある場合、AIは実装せず質問します

AIは 補助意見であり、判断主体ではありません
